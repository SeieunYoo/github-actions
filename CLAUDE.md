# CLAUDE.md — 숏츠 자동화 작업 가이드

이 레포는 **YouTube Shorts 자동 생산 시스템**을 만들기 위한 공간입니다.
Claude Code가 이 문서를 먼저 읽고 작업 컨텍스트를 잡습니다.

---

## 1. 작업할 때 항상 먼저 할 일

1. `docs/shorts-instruction-template.md` 를 읽고 **현재 시리즈의 지침**이 정의돼 있는지 확인
   - 정의되어 있지 않으면 사용자에게 캐릭터/세계관/톤을 먼저 물어볼 것
2. `docs/workflow.md` 의 단계대로 진행
3. 결과물(스크립트, 프롬프트, 영상)은 `output/<시리즈명>/<에피소드번호>/` 에 저장

---

## 2. 디렉토리 구조

```
.
├── CLAUDE.md                       # 이 문서
├── docs/
│   ├── shorts-instruction-template.md   # 시리즈 지침 템플릿 (캐릭터/세계관/톤)
│   └── workflow.md                      # 숏츠 1편 생산 파이프라인
├── prompts/                        # VEO3 / 이미지 모델용 프롬프트 빌더
├── scripts/                        # 자동화 스크립트 (ffmpeg, 합성, 업로드)
└── output/                         # 생성된 결과물 (gitignore 권장)
```

## 3. 핵심 원칙

- **캐릭터 일관성이 최우선**: 모든 장면 프롬프트에 캐릭터 시트의 고정 묘사를 반드시 포함
- **장면 단위 분해**: VEO3는 한 번에 8초 내외 → 숏츠 1편(45-60초)은 6-8개 클립으로 쪼갬
- **9:16 세로 비율** 고정, 1080x1920 권장
- **실패하면 재시도 말고 원인 분석**: 캐릭터가 달라지면 프롬프트 시드/묘사를 먼저 의심

## 4. 작업 시 체크리스트

- [ ] 시리즈 지침 파일이 존재하는가
- [ ] 캐릭터 시트의 필수 항목(외형/의상/말투)이 모두 채워져 있는가
- [ ] 장면별 프롬프트에 캐릭터 시트 묘사가 그대로 들어가 있는가
- [ ] 결과 영상이 9:16, 60초 이내인가
- [ ] 자막/훅(첫 3초)이 들어가 있는가

## 5. 외부 도구 / API

| 용도 | 도구 | 비고 |
|---|---|---|
| 영상 생성 | VEO3 (Google Vertex AI / AI Studio) | API 키 필요 |
| 이미지 생성 | Imagen / Midjourney / SDXL | 캐릭터 시트 레퍼런스용 |
| 음성/TTS | ElevenLabs, OpenAI TTS | 내레이션 |
| 영상 합성 | ffmpeg | 클립 이어붙이기, 자막, BGM |
| 자막 | Whisper (STT) → SRT | 자동 자막 |

API 키는 `.env` 에 두고 절대 커밋하지 않습니다.

## 6. 사용자와의 소통

- 사용자가 "숏츠 만들어줘" 라고 하면 → 시리즈 지침이 있는지 먼저 확인하고, 없으면 템플릿을 채우도록 안내
- 사용자가 주제만 던지면 → 지침에 정의된 캐릭터/톤으로 스크립트를 작성
- 모호하면 추측하지 말고 물어볼 것 (특히 캐릭터 외형)
