# 숏츠 1편 생산 워크플로우

> Claude Code가 "숏츠 한 편 만들어줘" 요청을 받았을 때 따르는 표준 절차.

---

## 0. 사전 확인

- [ ] `docs/series/<시리즈명>.md` 시리즈 지침이 존재하는가?
  - 없으면: `docs/shorts-instruction-template.md` 를 복사해서 사용자와 함께 채운 후 진행
- [ ] 이번 에피소드의 **주제/메시지** 가 명확한가?
  - 없으면: 사용자에게 한 문장으로 받기

---

## 1. 기획 (1-2분)

1. 시리즈 지침의 캐릭터 시트 / 톤 / 포맷을 컨텍스트로 로드
2. 에피소드 한 줄 요약 → 훅(첫 3초) 후보 3개 생성
3. 사용자 선택 또는 자동 선택

**산출물**: `output/<시리즈>/<ep번호>/01_brief.md`

---

## 2. 대본 작성 (2-3분)

1. 시리즈 톤에 맞춰 대본 작성 (구간별 시간 배분 준수)
2. 대사 / 내레이션 / 화면 지시 분리
3. 자막용 텍스트 별도 추출

**산출물**:
- `output/<시리즈>/<ep번호>/02_script.md` (사람이 읽기 좋은 형태)
- `output/<시리즈>/<ep번호>/02_script.json` (장면 단위 구조화)

`script.json` 스키마:
```json
{
  "episode": 1,
  "title": "...",
  "scenes": [
    {
      "id": 1,
      "duration_sec": 3,
      "narration": "...",
      "subtitle": "...",
      "visual_direction": "캐릭터가 카페 창가에 앉아 있다, 빛은 오후 햇살",
      "camera": "slow zoom in"
    }
  ]
}
```

---

## 3. 비주얼 프롬프트 생성 (1분)

각 장면(`scene`) 에 대해 VEO3 프롬프트를 만든다.

**프롬프트 구성 규칙** (이 순서를 지킬 것):
1. `[CHARACTER_BLOCK]` — 시리즈 지침의 캐릭터 고정 묘사 블록 (그대로 복붙)
2. `[BACKGROUND_BLOCK]` — 시리즈 지침의 배경 고정 묘사 블록
3. `[ACTION]` — 이 장면의 동작 (script.json 의 visual_direction)
4. `[CAMERA]` — 카메라 워크
5. `[STYLE]` — 스타일/색감/조명
6. `[NEGATIVE]` — 금지 사항

**산출물**: `output/<시리즈>/<ep번호>/03_prompts/scene_01.txt` ...

---

## 4. 영상 클립 생성 (5-10분, 네트워크 의존)

1. 각 프롬프트로 VEO3 호출 (MCP 또는 직접 API)
2. 결과 클립을 `output/<시리즈>/<ep번호>/04_clips/scene_01.mp4` 로 저장
3. **검수**: 캐릭터 일관성 / 배경 일관성 / 길이
   - 실패 장면은 시드만 바꿔 재생성 (프롬프트는 함부로 수정하지 말 것)

---

## 5. 음성 합성 (1분)

1. `script.json` 의 `narration` 을 TTS (ElevenLabs 등) 로 변환
2. `output/<시리즈>/<ep번호>/05_audio/scene_01.mp3` 로 저장

---

## 6. 합성 (1-2분)

`scripts/compose.sh` 또는 ffmpeg 명령으로:
1. 클립 이어붙이기 (concat)
2. 내레이션 오버레이
3. BGM 믹스 (보이스 -12dB, 음악 -20dB 권장)
4. 자막 burn-in (또는 별도 SRT 파일)
5. 워터마크/엔드카드

**산출물**: `output/<시리즈>/<ep번호>/final.mp4`

---

## 7. 검수 체크리스트

- [ ] 9:16, 1080x1920
- [ ] 길이 ≤ 60초
- [ ] 첫 3초에 훅이 있는가
- [ ] 캐릭터 외형이 모든 장면에서 동일한가
- [ ] 자막 가독성 OK
- [ ] BGM 볼륨이 보이스를 덮지 않는가
- [ ] CTA 포함 여부

---

## 8. 업로드 (선택)

- YouTube Data API 또는 수동 업로드
- 제목/설명/해시태그도 `output/<시리즈>/<ep번호>/06_meta.md` 에 미리 생성해두기

---

## 자주 발생하는 에러 / 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| 캐릭터 얼굴이 장면마다 다름 | 캐릭터 블록을 일부만 넣음 | 고정 묘사 블록 **전체** 를 모든 프롬프트에 복붙 |
| 배경이 갑자기 바뀜 | 배경 블록 누락 | 배경 블록도 모든 프롬프트에 포함 |
| 영상이 9:16이 아님 | API 파라미터 누락 | `aspect_ratio: "9:16"` 명시 |
| 길이 초과 | 장면 수 과다 | 6-8개로 제한, 한 장면 당 6-8초 |
| 자막이 화면 밖 | safe area 미고려 | 상하 240px 안쪽에 배치 |
